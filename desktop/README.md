# 订货特工 Guimarães · 桌面挂件

一个透明、无边框、**永远置顶**的桌面陪伴小人，停在屏幕右下角。
平时只显示一个像素小人；有订货提示时它会冒气泡、弹跳；点击展开订货面板
（订货建议 / 库存 / 配送日历 / 状态）。数据来自寿司订货系统 `sushi/orders`。

---

## 一、先跑起来（开发模式）

需要先装 [Node.js](https://nodejs.org)（18 以上）。然后在本文件夹里：

```bash
cd desktop
npm install        # 安装 Electron（第一次会下点东西，耐心等）
npm start          # 启动桌面挂件
```

小人就会出现在桌面右下角。
- **点小人** → 展开 / 收起订货面板（窗口会自动放大缩小）
- **拖面板顶部** → 移动整个挂件窗口
- **托盘图标**（菜单栏 / 状态栏）→ 显示·隐藏 / 退出

## 二、打包成安装程序

```bash
npm run dist
```

产物在 `dist/` 里：
- Windows → `.exe` 安装包
- macOS → `.dmg`
- Linux → `.AppImage`

（在哪个系统上打包，就出哪个系统的包。要出 Win 包一般在 Windows 上跑。）

---

## 三、文件说明

| 文件 | 作用 |
|---|---|
| `main.js` | Electron 主进程：透明置顶窗口、右下角锚定、展开时自动放大、托盘菜单 |
| `preload.js` | 安全桥：让网页通知主进程「展开/收起」 |
| `companion.html` | 整个应用（已把代码+设计系统+依赖打包进单文件，离线可用） |
| `package.json` | 依赖与打包配置 |

> `companion.html` 是从设计稿编译出来的产物，**不要直接改它**。
> 要改界面/逻辑，去改源工程（`index.html` + `app/`），再重新编译覆盖这里。

## 四、关于「真正抓取订货网站」——下一步

目前 `companion.html` 里的库存/订货数据是**贴合你真实数据的样本**（见源工程
`app/data.js`），订货引擎（`app/engine.js`）已经是真逻辑。要让它**自动登录并抓取**
`http://115.29.195.37:3001/sushi/orders` 的实时数据，在 Electron 里加一段抓取即可——
主进程能直接发网络请求、且能访问你的内网，正好补上浏览器做不到的那一环：

1. 在 `main.js` 里用 `net`/`fetch` 登录（root/root）拿会话；
2. 拉「库存统计」和「采购订单」两个接口，整理成 `app/data.js` 里的
   `inventory[]` / `calendar` 结构；
3. 通过 `preload.js` 把数据推给页面（替换 `window.APP_DATA`），
   每隔几分钟轮询一次即可。
